Research-Grade Matching (v2.0)

ShlokaAI v2.0 uses three matching modes designed for Ayurveda text mining:

  • Strict (default): Unicode-aware word boundaries + exact token matching. Prevents false positives like matching "ama" inside "samana".
  • Compound: Dictionary-based segmentation for sandhi/samasa forms (e.g. finds "vata" and "pitta" in "vatapittau").
  • Legacy: Original substring matching — useful for comparison only, not recommended for publication.

For full morphological decompounding (all sandhi rules), integrate external analyzers such as the INRIA Sanskrit Reader or Sanskrit Heritage API in future pipeline stages.

How to Use ShlokaAI

  1. Load templates or define concepts with Devanagari/IAST synonyms.
  2. Choose matching mode and false-positive guards under Analysis Settings.
  3. Paste or upload text, run analysis, then export wide CSV (per-concept columns) or JSON with full metadata.
